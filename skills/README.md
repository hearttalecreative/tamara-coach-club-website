# Skills del proyecto

Estructura base para organizar skills reutilizables durante el desarrollo del website.

## Convencion sugerida

- Cada skill vive en su carpeta numerada.
- Archivo principal: `skill.md`.
- Soporte opcional: `assets/`, `checklists/`, `prompts/`, `references/`.

## Skills iniciales

- `00-shared`: reglas globales del proyecto
- `01-brand-strategy`: posicionamiento y mensaje
- `02-ui-visual`: sistema visual y componentes UI
- `03-landing-layout`: estructura one-page y jerarquia
- `04-frontend-development`: implementacion HTML/CSS/JS
- `05-css-animations`: animaciones CSS
- `06-web-animation-interactivity`: interactividad avanzada
- `07-user-experience`: UX y flujo de conversion
- `08-forms-brevo`: formularios e integracion con Brevo
- `09-analytics`: analitica y eventos
- `10-performance`: rendimiento web
- `11-content-marketing`: SEO local y contenido
- `12-strategic-marketing-design`: alineacion entre marketing y direccion visual

## Integracion de HABILIDADES

Se incorporaron los documentos de `HABILIDADES/*.docx` como base metodologica.
Los textos convertidos estan en `skills/source-habilidades/` y cada skill incluye su mapeo en `references/sources.md`.

Usa `templates/skill-template.md` para crear nuevas skills.
